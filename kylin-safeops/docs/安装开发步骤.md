# 安装开发步骤

## 1. 开发模式说明

项目按 Linux/Kylin 兼容模式开发，运行模式由 `SAFEOPS_MODE` 控制：

```text
demo：Windows/Docker 开发，使用可控样例数据
real：Kylin/openKylin 验证，调用真实 systemctl/journalctl/ss/ps
auto：自动探测，Linux 且工具齐全时走 real，否则走 demo
```

推荐：

```text
Windows 本机：demo
Docker 联调：demo
Kylin/openKylin：real
```

## 2. Windows 本机开发

最简单方式：不用管虚拟环境，直接执行：

```powershell
cd D:\content\bs\kylin-safeops
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 start
```

它会自动完成：

```text
1. 如果没有 .venv，就创建 Python 隔离环境
2. 安装后端依赖
3. 如果没有 node_modules，就安装前端依赖
4. 后台启动后端 8000
5. 后台启动前端 5173
6. 如果 8000/5173 已经有 SafeOps 服务在跑，就直接复用
```

查看状态：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 status
```

停止：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop_dev.ps1
```

停止脚本会优先停止脚本记录的 PID；如果没有 PID 文件，也会检查 `8000` 和 `5173` 上看起来属于 SafeOps 的进程。

如果你想手动开两个终端，也可以用下面的原始方式。

后端：

```powershell
cd D:\content\bs\kylin-safeops
powershell -ExecutionPolicy Bypass -File .\scripts\start_backend_demo.ps1
```

前端另开一个终端：

```powershell
cd D:\content\bs\kylin-safeops
powershell -ExecutionPolicy Bypass -File .\scripts\start_frontend.ps1
```

访问：

```text
前端：http://localhost:5173
后端：http://localhost:8000
环境探测：http://localhost:8000/api/environment/probe
```

## 3. Docker 开发

需要安装 Docker Desktop，并确保 `docker compose` 可用。

```powershell
cd D:\content\bs\kylin-safeops
docker compose up --build
```

Docker 默认使用 `SAFEOPS_MODE=demo`，只用于联调，不作为最终麒麟证明环境。

## 4. Kylin/openKylin 验证

复制代码到 Kylin/openKylin 后：

```bash
cd kylin-safeops
bash scripts/check_kylin_compat.sh
bash scripts/start_backend_kylin.sh
```

前端：

```bash
bash scripts/start_frontend_kylin.sh
```

确认接口：

```bash
curl http://localhost:8000/api/environment/probe
```

页面中应显示：

```text
effective_mode: real
systemctl: true
journalctl: true
ss: true
ps: true
```

注意：如果手动设置 `SAFEOPS_MODE=real`，但当前环境不是 Linux/systemd 或缺少必要工具，系统会明确返回 real 模式不可用，不会静默使用 demo 数据。这是为了防止最终验证材料误判。

## 5. DeepSeek 配置

不要把真实 Key 写进代码或文档。

Windows PowerShell：

```powershell
$env:DEEPSEEK_API_KEY="你的真实 Key"
$env:DEEPSEEK_ENABLED="true"
```

Linux/Kylin：

```bash
export DEEPSEEK_API_KEY="你的真实 Key"
export DEEPSEEK_ENABLED=true
```

默认 `DEEPSEEK_ENABLED=false`，系统自动使用规则兜底。只有显式设置为 `true` 且提供 `DEEPSEEK_API_KEY` 时，后端才会访问 DeepSeek。

## 6.1 不懂虚拟环境也没关系

`.venv` 可以理解成：

```text
项目自己的 Python 工具箱
```

你不需要手动激活它，也不需要记命令。只要用：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 start
```

脚本会自动调用 `.venv\Scripts\python.exe`。这样不会污染你电脑上的全局 Python，也更适合迁移到 Kylin/openKylin 后重新创建 Linux 版本 `.venv`。
