# 开发说明

## Docker 与麒麟环境的关系

Docker 不等同于 KylinOS。Docker 容器共享宿主机内核，容器内的 `systemctl`、`journalctl`、服务管理能力和真实 KylinOS 可能不同。

推荐路线：

```text
本机 Docker：开发、前后端联调、规则流程验证
openKylin/KylinOS：最终真实工具验证、截图、录屏
```

## 当前工程状态

当前是可启动骨架：

- 后端 FastAPI。
- 前端 React。
- PlanSpec 示例。
- Knowledge State 示例。
- Docker Compose。
- `SAFEOPS_MODE=demo|real|auto` 兼容模式。
- 环境探测接口 `/api/environment/probe`。

下一步：

1. 在 openKylin/KylinOS 上用 `SAFEOPS_MODE=real` 验证工具输出。
2. 根据真实输出微调解析规则。
3. 接入 DeepSeek Cognitive Critic。
4. 打磨 Counterfactual Evidence Graph 前端可视化。
5. 增加 Replay 和 Audit 存储。

## 模式配置

```text
SAFEOPS_MODE=demo
```

适合 Windows/Docker 开发。

```text
SAFEOPS_MODE=real
```

适合 openKylin/KylinOS 最终验证。

```text
SAFEOPS_MODE=auto
```

自动探测。Linux 且工具齐全时走 real，否则走 demo。
