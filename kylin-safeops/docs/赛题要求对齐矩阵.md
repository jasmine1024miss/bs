## 赛题定位

赛题 1：A-面向麒麟操作系统的安全智能运维 Agent 设计与实现。

项目定位：KylinSafeOps 是部署于麒麟/Linux 环境的常驻式安全可信智能运维 Agent。系统以自动巡检和事件触发为主入口，持续感知服务、端口、进程、日志和资源状态；发现异常后主动告警、自动生成可验证 PlanSpec、调用受控 OS 工具采集证据、输出证据图谱、根因结论与审计报告。自然语言输入保留为追问、解释和高影响处置确认入口。

## 当前最新诊断契约

一次由自动巡检或人工追问触发的 nginx 启动失败诊断会返回：

- PlanSpec：6 步
- Tool Trace：`systemctl_status`、`journalctl_unit`、`ss_listen`、`netstat_listen`、`lsof_port`、`ps_process`
- Evidence Graph：包含日志现象、端口状态、netstat 交叉验证、lsof 端口归属、进程归属、根因和建议节点
- Root Cause：端口冲突导致 nginx 启动失败
- Audit ID：持久化审计会话，可在审计中心回放和导出
- Alert Event：异常事件 ID、风险等级、处理状态和告警来源

## 仍需最后验证

1. 在 KylinOS/openKylin 环境运行 `SAFEOPS_MODE=auto`，确认自动进入 `real` 或明确说明真实工具阻断原因。
2. 录制真实环境运行视频，展示 `journalctl / netstat / lsof / ps` 工具轨迹。
3. 展示非 root 或受限账户运行方式，补足“最小权限执行”的现场证据。
4. 在答辩材料中说明：当前实现采用 Tool Contract + Adapter 的 MCP-style 插件化架构，若要求严格 MCP，可再加 MCP server 包装层。
5. 展示 Agent 常驻运行、自动巡检、异常告警弹窗和自动生成审计会话的截图或录屏。
