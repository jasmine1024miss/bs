# 不用懂虚拟环境的开发方式

## 一句话

你只需要运行：

```powershell
cd D:\content\bs\kylin-safeops
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 start
```

然后打开：

```text
http://127.0.0.1:5173/
```

## 虚拟环境是什么

`.venv` 就是这个项目自己的 Python 工具箱。

它的作用是：

```text
不污染你的电脑
不影响别的项目
让后端依赖固定在本项目里
```

你不用手动打开它。

## 常用命令

检查：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 check
```

安装依赖：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 install
```

启动前后端：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 start
```

查看状态：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 status
```

如果 `5173` 或 `8000` 已经有 SafeOps 服务在运行，启动脚本会直接复用，不会重复开一份。

停止：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop_dev.ps1
```

停止脚本会优先停止脚本自己启动的进程；如果没有 PID 文件，也会检查 `5173` 和 `8000` 上看起来属于本项目的进程。

## Kylin/openKylin 上怎么做

到了 Kylin/openKylin 后，不复制 Windows 的 `.venv`。

在 Kylin/openKylin 里重新执行：

```bash
bash scripts/kylin_install_runtime.sh
SAFEOPS_MODE=real bash scripts/start_backend_kylin.sh
bash scripts/start_frontend_kylin.sh
```

这样会创建 Linux/Kylin 自己的 `.venv`，避免 Windows 和 Linux 依赖不兼容。
